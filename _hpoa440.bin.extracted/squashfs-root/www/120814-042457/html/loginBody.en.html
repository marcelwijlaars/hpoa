Which browsers are supported?
Requirements
The HP BladeSystem Onboard Administrator Web interface requires an XSLT enabled browser with support for JavaScript® 1.3 or equivalent.

If you received a notice that your browser does not have the required functionality, examine your browser settings to see if they meet the requirements below or contact your administrator.

Known Browser Issues
If you are using Internet Explorer 8 or above, you may have to add the Onboard Administrator's domain to your trusted sites for the application to work properly. The Onboard Administrator web interface is only supported when running Internet Explorer 8 in compatibility mode. Consult Internet Explorer Help for information on these topics.

If you are using Firefox 3.6 or above, pressing the ESC key disconnects any open XMLHttpRequest connections, resulting in unexpected client behavior. Consult the Mozilla documentation for more information (issue 614304).

Javascript
Client-side Javascript is used extensively by this application. Check your browser settings to make sure javascript is enabled before running the application.

ActiveX
When using Microsoft® Internet Explorer with this application, ActiveX must be enabled. Check your browser settings to make sure ActiveX is enabled before running the application.

Popup Windows
Popup windows must be allowed for certain features to function correctly. Check your browser settings to make sure popup blockers are not enabled before running the application.

Cookies
Cookies must be enabled for certain features to function correctly. Check your browser settings to make sure cookies are enabled before running the application.

What is a cookie?
A cookie is small, unscriptable text file placed on your hard drive by a web site to preserve specific settings. When you return to the site, your computer serves up the "cookie" with the previously saved settings so they can be passed along to the site. Cookies are also used to temporarily store session data.

How do I sign in?
Access to the application must be through HTTPS (HTTP exchanged over an SSL encrypted session). See 'What is SSL?' for more information about SSL.

Enter the user name and password for your account to HP BladeSystem Onboard Administrator.

If you have problems signing in, it could be for one of the following reasons. Contact the administrator if you continue to have problems.
  • You have recently upgraded the Onboard Administrator's firmware. You may have to clear your browser's cache before attempting to login again.
  • You are not entering the information correctly. Passwords are case-sensitive.
  • The account you are entering is not an account for HP BladeSystem Onboard Administrator.
  • The account you are entering has been deleted, disabled, or locked out.
  • The password for the account needs to be changed.
  • You are attempting to sign in from an IP address that is not valid for the specified account.
Selecting Enclosures
All enclosures connected via your primary enclosure's interlink ports are displayed in the rack topology table above. The primary enclosure is selected by default, and cannot be deselected.

When linked enclosures are displayed, the topology mode used by the application during your session is determined by the checkbox selections made before signing in, as described below:
  • Local Mode - this is the default topology mode, and is enabled if none of the linked enclosures are selected. All topology changes that occur while signed in will be ignored by the application.
  • Fixed Mode - this topology mode is enabled when some of the linked enclosures are selected, but not all of them. Only topology changes that affect the selected enclosures while signed in will be displayed by the application.
  • Linked Mode - this topology mode is enabled if all displayed enclosures are selected. All topology changes that occur while signed in will be displayed by the application.
The application will attempt to sign in to each of the linked enclosures that you selected, and will load enclosure information for each of the authenticated enclosures. Load times will vary depending on the speed of your connection. For slow connections, it is recommended that only the primary enclosure be selected when signing in.

Note: Full topology (Linked Mode) can be enabled from within the application by clicking the Refresh Topology button located on the Rack Overview page.

What is SSL?
Secure Sockets Layer (SSL) is a standard for encrypting data so that it cannot be viewed or modified by others while in transit on the network. SSL uses a key to encrypt and decrypt the data. The longer the key, the better the encryption. HP Onboard Administrator supports up to 128-bit length keys.

What is a certificate?
A certificate is a public document that describes the server so it can be identified. It contains the name of the server and also the server's public key. Since only the server has the corresponding private key, this is how the server is authenticated.

A certificate must be signed to be valid. If it is signed by a Certifying Authority (CA) and that CA is trusted, all certificates signed by the CA are also trusted. A self-signed certificate is one that is signed by itself. In other words, the owner of the certificate is also acting as its own CA. Self-signed certificates are the default for HP management products, though they do support certificates signed by Certifying Authorities.

How do I handle an Unknown Authority?
When the "Website Certified by an Unknown Authority" pop-up is displayed, take the following action:
  1. Ensure that you are browsing to the desired management server (not an imposter):
    • View the certificate.
    • Ensure that the "Issued To" name is your management server. Perform any other steps you feel necessary to verify the identity of the management server.
    • If you are not sure this is the desired management server, do not proceed. You could be browsing to an imposter, and subsequently giving your sign-in credentials to that imposter when you sign in. Contact the administrator. Exit the certificate window and click No or Cancel to cancel the connection.
  2. If the above check is successful, you have several options:
    • Accept the certificate temporarily for this session.
    • Accept the certificate permanently.
    • Stop now and import the certificate into your browser from a file provided by your administrator.
